# ConfigBuddy v3.0 Expansion - Implementation Plan

**Status:** Ready for Implementation
**Created:** 2025-11-05
**Target Completion:** 12 months (4 phases)
**Based on:** `/design/architecture/cmdb_v3_unified.txt`

---

## Executive Summary

ConfigBuddy v3.0 represents a major architectural evolution that unifies **ITIL v4 Service Management**, **TBM v5.0.1 Cost Transparency**, and **Business Service Mapping** into a single cohesive platform. This expansion builds on the solid v2.0 connector framework foundation and extends it with business context, financial transparency, and strategic value mapping.

### Key Objectives

1. **Unified Data Model**: Extend CIs with ITIL, TBM, and BSM attributes
2. **Cost Transparency**: Implement full TBM cost allocation from infrastructure to business capabilities
3. **Business Value Mapping**: Connect IT infrastructure to business outcomes
4. **Advanced Analytics**: Multi-perspective dashboards for different stakeholder groups
5. **Integration Services**: Unify ITIL, TBM, and BSM operations through single API

### Success Criteria

- ✅ All v2.0 functionality preserved and working
- ✅ 100% Tier 0/1 business services mapped
- ✅ 95%+ cost allocation accuracy
- ✅ 90%+ configuration accuracy
- ✅ <5 minute blast radius analysis
- ✅ Support for 1M+ CIs with sub-second query performance

## Architecture Review

### Current State (v2.0)

**Strengths:**
- ✅ Solid connector framework (38 connectors: 18 TypeScript + 20 JSON)
- ✅ Discovery engine with agent-based architecture
- ✅ Neo4j graph database for relationships
- ✅ PostgreSQL data mart for analytics
- ✅ Unified credential system
- ✅ Dynamic metadata support

**v2.0 Stack:**
- Node.js 20 LTS + TypeScript 5.x
- Neo4j Community v5.x (graph)
- PostgreSQL 15+ with TimescaleDB (data mart)
- Redis v7.x (cache/queue)
- BullMQ v4.x (job orchestration)
- React 18 + Vite (UI)

### Target State (v3.0)

**New Capabilities:**
- 🆕 ITIL v4 service configuration management
- 🆕 TBM v5.0.1 cost allocation engine
- 🆕 Business service mapping and impact scoring
- 🆕 Unified service interface (ITIL + TBM + BSM)
- 🆕 Multi-stakeholder dashboards
- 🆕 Advanced analytics and BI integration

**Additional Stack Components:**
- Apache Kafka v3.x (event streaming for real-time updates)
- Metabase (self-service BI dashboards)
- Enhanced Grafana dashboards (operations metrics)

### Gap Analysis

| Component | v2.0 Status | v3.0 Required | Effort |
|-----------|------------|---------------|--------|
| **Data Model** | CI-focused | CI + Business entities | HIGH |
| **Discovery Engine** | Infrastructure-only | Infrastructure + Business context | MEDIUM |
| **Cost Allocation** | Not present | Full TBM engine | HIGH |
| **Impact Scoring** | Not present | Business impact calculator | MEDIUM |
| **Unified API** | Separate endpoints | Single unified interface | MEDIUM |
| **Dashboards** | CMDB-only | Multi-stakeholder views | HIGH |
| **Event Streaming** | BullMQ only | Kafka + BullMQ | MEDIUM |
| **BI Integration** | Limited | Full Metabase integration | LOW |

## Migration Strategy

### Zero-Downtime Deployment

1. **Phase 1-3**: Deploy alongside v2.0
   - New packages deployed but not active
   - Database migrations applied (backward compatible)
   - Feature flags control v3.0 activation

2. **Phase 4**: Gradual rollout
   - Enable v3.0 features for test business services
   - Monitor performance and accuracy
   - Collect user feedback

3. **Final Cutover**: Complete transition
   - Enable all v3.0 features
   - Redirect dashboards to v3.0 views
   - Deprecate v2.0-only endpoints

### Rollback Plan

- All database migrations have rollback scripts
- Feature flags allow instant disable of v3.0 features
- v2.0 functionality preserved throughout
